## 2. 已识别问题清单（执行口径）

### P0（优先修复）
1. 高价值线索接口返回明文联系方式，存在绕平台私联成交风险。

2. “仲裁期间冻结交易”仅有前端文案，后端缺少硬性拦截。
   - 风险：争议期间仍可私下推进状态，平台规则失效。
3. 意向接受到自动成单存在并发重复生成风险。
   - 风险：高并发下可出现重复成单或状态竞争。

### P2（后续）

## 3. 执行约束（Agent 必须遵守）
1. 修复前先阅读相关文档与现有实现，避免偏离业务。
2. 不做破坏性操作：不重置仓库、不回滚用户历史改动。
3. 每次改造后必须做最小可验证检查（语法/错误检查 + 关键路径验证）。
4. 文档与代码同步更新，保持可追溯。

## 4. P0 执行计划
1. 接口脱敏收敛：默认脱敏输出 phone，管理端或所有者按权限查看明文。
2. 后端冻结落地：对关键状态流转接口增加“活跃仲裁”拦截。
3. 成单并发加固：接受意向时先原子抢占状态，再执行建单。

## 5. 验收标准（P0）
1. 非管理员视角下，线索接口不再返回明文电话。
2. 当目标处于 `pending/investigating` 仲裁中时，关键状态变更被后端拒绝（409）。
3. 并发接受同一意向时，仅允许一次成功成单，其余请求返回冲突。
4. 代码通过基础错误检查，且无新增语法错误。
